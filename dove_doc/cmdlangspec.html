Intro

The Commandline is one of the most central features of this Toolcollection. All operations including editing the configuration can aswell be executed via the commandline. This section of the documentation describes the syntax of Commandline commands. This commandline supports low-level OOP. This means, the user can create structures and methods and use them to solve more complex problems.

Scripts

The CommandLine can also execute scripts (run -s scriptfile)

Datatypes

The commandline provides several datatypes. By default the datatype is specified implicitly by the syntax. Nevertheless there are several operations that require specifying the datatype aswell as some special cases.

  • Text

    Text can be created in this way: "some text"

  • Integer

    The syntax for integral numbers is: 12415

  • Double

    The syntax for real numbers is: 1241.234

  • Boolean

    Truthvalues have several representations: Either, if specified in numbers, 0 is false and 1 true, or directly via true and false.

There also exist specific operations for each datatypes. These will be discussed later on in separate paragraphs.

Variables

This paragraph describes the creation and usage of variables. The variable structure in the commandline provides the following options: variables can be made typesafe. And they can be secured to not be overwritten by newer values. Creating a new variable: 
                                 set a="hello world"
This code generates a new variable - if it doesn't yet exist - named a and overwrites it's content with the text "hello world". value. Reading values: 
                                 set -u a="Please enter a value"
This code displays the message "Please enter a value" and afterwards sets a to the new value entered by the user. The end of the input is specified by pressing ENTER. Making a variable typesafe: 
                                 set -typesafe=true a=<...>
makes the variable a typesafe (Note: <...> replaces an optional argument . This means: there can't be assigned any other type of value to this variable without updating the typesafety. For example: 
                                 set -typesafe=true a=123
                                 set a="hello world"
will cause an error, since a is locked to only hold integral values. Due to this assigning the textvalue "hello world" is an invalid operation. The lock can be removed by calling set -typesafe=false a=<...> Making a variable unwriteable: 
                                 set -final=true a=<...>
After this operation, the value can only be read. Writing operations will be quit with an errormessage. This can be undone by set -final=false a=<...> So far, this section has only discussed creating variables, assigning values to them and changing their state. So last but not least, the most important feature of variables: reading 
                                 a
That's it. Or more extended: 
                                 set a="hello world"
                                 print(a)
will print "hello world" to the console (the print command will be discussed in one of the next paragraph). The last notes: the attributes of variables can be read using the meta-operator: 
                                 set a=
                                 print $a.isFinal()       //read the final-state of the variable
                                 print $a.isTypesafe()    //read typesafetyattribute
                                 print $a.val()         //just another way to read the value
The meta-operator ( 
$
) allows access to some of the meta-data of a variable, like type, typesafety, overwritability of the variable, etc. This will be discussed in detail later on.
